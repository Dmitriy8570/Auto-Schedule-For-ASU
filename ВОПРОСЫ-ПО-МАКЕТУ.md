та же навигация и вкладки; «Нагрузка» и «История» подключены к бэкенду). То есть это не
переписывание с нуля, а закрытие оставшихся пробелов (~15–20%).

## 1. Фреймворк фронтенда: оставляем Vue или переходим на React (как в макете)?

Макет `figma/` написан на **React + Tailwind v4 + Radix UI**. Текущий проект — **Vue 3 + TS**.
По контексту диплома фронтенд ведёт напарник именно на Vue, и в нём уже есть рабочая
интеграция с API (auth, нагрузка, история, справочники).

- [+] **(рекомендую)** Оставить Vue, доводить его до макета по дизайну/логике.
      Причина: сохраняем рабочую интеграцию и стек команды; «как написано — неважно»
      (твои слова), важны дизайн и логика — они уже совпадают.
- [ ] Перейти на React-приложение из `figma/`, заново подключить API-слой.
      Даёт пиксель-в-пиксель макет, но выкидывает Vue-работу напарника и интеграцию.

> Пока действую по варианту 1 (Vue).

## 2. Сетка расписания: дорабатываем бэкенд, чтобы занятия рисовались в сетке?

Сейчас `GET /api/lessons/by-*` отдаёт «сырой» `LessonView` (только GUID'ы: classroomId,
timeSlotId, streamId, version). Фронт не может разложить занятия по сетке «дни × пары»,
потому что нет:
- дня недели и номера пары (нужен справочник `TimeSlot` → день + пара + время);
- отображаемых полей (дисциплина, ФИО преподавателя, группа, аудитория, тип занятия);
- сопоставления недели (UI оперирует номером 1..18, а `weekId` — GUID).

Это центральный пробел до макета, но он в **бэкенде (твоя зона)** и затрагивает доменную
модель/солвер, поэтому не трогаю без подтверждения.

- [+] **(рекомендую)** Да, обогатить `LessonView` (день, пара, времена, дисциплина, ФИО,
      группа, аудитория, тип, чётность недели Red/Blue) + добавить `GET /api/timeslots`,
      `GET /api/semesters`, `GET /api/weeks?semesterId=`. Тогда сетка заработает.
- [ ] Сделаю бэкенд сам — фронт подготовь под уже описанный в `endpoints.txt` контракт.
- [ ] Пока оставить заглушку (как сейчас: «Найдено занятий: N»).

Если вариант 1 — уточни: реализую сам или ты предпочтёшь вести доменную часть лично
(тогда дай зелёный свет на правку query-хендлеров и репозиториев расписания).

## 3. Редактирование расписания (панель «Добавить/Изменить пару»)

В макете справа есть панель добавления/редактирования пары (дисциплина, тип, преподаватель,
группа, аудитория) + перемещение и удаление пары.

Бэкенд: `POST /api/lessons` принимает `{classroomId, timeSlotId, streamId, semesterId}` —
но UI оперирует «дисциплина+преподаватель+группа», а это `streamId/curriculumId`; маппинга
нет. Эндпоинтов `DELETE`/обновления занятия тоже нет.

- [+] Реализовать полноценно (добавить недостающие эндпоинты + маппинг).

      только когда п.2 закрыт (иначе кнопки-заглушки с тостом «появится после интеграции»).
- [ ] Не делать панель редактирования сейчас.

## РЕАЛИЗОВАНО по твоим ответам (бэкенд + Vue, всё собирается, тесты зелёные)

**1. Стек — Vue.** Оставлен Vue, доведён до макета.

**2. Сетка расписания (бэкенд + фронт).**
- `Lesson` получил `CurriculumId` (миграция `AddLessonCurriculumId`, FK SetNull) — теперь у
  занятия известны дисциплина/преподаватель/тип. Солвер-маппер проставляет план.
- `LessonView` обогащён: день, пара, тип недели Red/Blue, дисциплина, ФИО, группы, аудитория,
  корпус, тип занятия. Репозиторий грузит навигации, хендлеры мапят через `LessonDTO.From`.
- Новые эндпоинты: `GET /api/semesters`, `GET /api/weeks?semesterId=`,
  `GET /api/timeslots?weekId=`, `GET /api/curriculums?...`.
- `ScheduleTab.vue`: выбор семестра + реальной недели, раскладка занятий по сетке с цветами по
  типу (лекция/практика/лаб/экзамен/консультация), генерация по институту вызывает солвер.

**3. Редактирование пары.** Панель добавления (день/пара/учебный план/аудитория) → `POST /api/lessons`
(с `curriculumId`); выбор пары → удаление (`DELETE /api/lessons/{id}`) и перемещение (пересоздание
в новой ячейке). Слоты ячеек резолвятся через `/api/timeslots`.

**4. Ограничения → «Объекты» (полный CRUD, бэкенд + фронт).**
- Аудитории: список/создание/редактирование/удаление (`/api/classrooms`).
- Оборудование: список/создание/удаление (`/api/equipments`).
- Веса мягких ограничений: чтение/изменение (`/api/constraints`) — новая под-вкладка «Веса».
- Левые деревья (препод./ауд./группы) пока остаются read-only моками — не входили в CRUD-объём;
  при желании свяжу со справочниками.

**5. Демо-блок на логине — не добавлял** (вход доменный).

> до выгруженного» (публикация/откат) пока визуальные.
