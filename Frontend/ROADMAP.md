## P0 — пробелы бэкенда, блокирующие сетку расписания

Без этих эндпоинтов `ScheduleTab` не может разложить занятия по сетке «дни × пары».

- [ ] **Обогатить отображение занятия.** Сейчас `LessonDTO` — только GUID'ы. Нужен
  «view»-DTO для расписания: день недели, номер пары, предмет, ФИО преподавателя,
  название аудитории, тип занятия, тип недели (Red/Blue). Варианты:
  - расширить ответ `GET /api/lessons/by-*`, либо
  - добавить `GET /api/schedule?entity=&id=&weekId=` со сформированными ячейками.
- [ ] **`GET /api/timeslots`** — справочник слотов (день недели + номер пары + время),
  чтобы фронт строил оси сетки и сопоставлял `timeSlotId` → ячейка.
- [ ] **`GET /api/semesters`** — список семестров (id, даты, активный). Нужен для:
  выбора семестра, генерации, фильтра нагрузки/истории по семестру.
- [ ] **`GET /api/weeks?semesterId=`** — недели семестра (id, номер, тип Red/Blue, даты).
  Сейчас UI оперирует номером 1..18, а `weekId` на бэке — GUID; нет сопоставления.

После P0: связать ось недель (`weekId`), отрисовать занятия в ячейках, бейдж типа недели.
