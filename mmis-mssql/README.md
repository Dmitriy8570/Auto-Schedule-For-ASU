# MMIS — имитация исходной БД MS SQL

Контейнер с **MS SQL Server 2022**, который имитирует базу системы **ММИС
«Деканат»** — источник, из которого Auto-Schedule импортирует справочники и
учебную нагрузку. При первом запуске автоматически создаётся схема и
загружаются фейковые данные на весь семестр.

## Что внутри

| Файл | Назначение |
|------|------------|
| `Dockerfile` | образ на базе `mcr.microsoft.com/mssql/server:2022-latest` |
| `entrypoint.sh` | запускает SQL Server + фоновую инициализацию |
| `configure-db.sh` | ждёт готовности сервера и применяет SQL-скрипты |
| `sql/01_schema.sql` | схема `mmis` (таблицы), идемпотентно |
| `sql/02_seed.sql` | процедурная генерация фейковых данных |
| `sql/03_views.sql` | представления с агрегированной нагрузкой |

## Объём данных

- **3** института, **9** кафедр, **36** преподавателей;
- **36** групп (4 курса × 3 группы × 3 института), 20–30 студентов;
- **28** дисциплин;
- **1 семестр = 18 недель** с чередованием красная/синяя;
- объём группы зависит от курса: **16 / 15 / 14 / 13 пар/неделю** (26–32 ч);
- суммарно **522 пары/неделю** распределены между преподавателями по точным
  целям 12–17 пар → **у каждого преподавателя 24–34 ч/неделю** (в норме 20–40),
  с равномерным разбросом по «ставкам»;
- семестровая нагрузка (`SemesterWorkloads`) — по каждому плану (432–612 ч на
  преподавателя за семестр);
- понедельная нагрузка (`WeekWorkloads`) — **9 396 строк** (522 плана × 18 недель).

Договорённость: **1 пара = 2 академических часа**. Каждая строка учебного плана
(`Curriculums`) = 1 пара/неделю; нагрузка преподавателя = сумма его строк плана.

## Схема (`mmis`)

```
Institutes ─< Departments ─< Teachers ─┐
Institutes ─< Degrees ─< Courses ─< Groups ─┐
Subjects ───────────────────────────────────┼─< Curriculums ─< SemesterWorkloads
Semesters ─< Weeks ──────────────────────────┘            └─< WeekWorkloads
```

`Curriculums` — учебный план (преподаватель + группа + дисциплина + тип
занятия + пар/неделю). От него идут семестровая и понедельная нагрузки.

## Представления

| Представление | Что показывает |
|---------------|----------------|
| `mmis.vGroupWeeklyLoad`     | пары и часы в неделю по группам |
| `mmis.vGroupSemesterLoad`   | часы за семестр по группам |
| `mmis.vTeacherWeeklyLoad`   | пары и часы в неделю по преподавателям |
| `mmis.vTeacherSemesterLoad` | часы за семестр по преподавателям |
| `mmis.vWorkloadDetails`     | плоская витрина всего плана |

## Запуск

Из корня проекта (поднимется вместе со всем стеком):

```bash
docker compose up --build mmis-db
```

Параметры берутся из `.env` (см. `.env.example`):
`MMIS_SA_PASSWORD`, `MMIS_PORT` (по умолчанию `1433`).

## Подключение и проверка

```bash
# логин: sa, пароль: $MMIS_SA_PASSWORD, база: MMIS
docker exec -it auto-schedule-mmis /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U sa -P "$MMIS_SA_PASSWORD" -C -d MMIS \
  -Q "SELECT TOP 10 * FROM mmis.vTeacherSemesterLoad ORDER BY SemesterHours DESC"
```

Снаружи (любой клиент: SSMS, Azure Data Studio, DBeaver):
`Server=localhost,1433` · `User=sa` · база `MMIS`.

> Данные хранятся в volume `mmis-data`. Скрипты идемпотентны: повторный запуск
> не дублирует данные. Чтобы пересоздать с нуля —
> `docker compose down -v` (удалит volume) и снова `up`.
