## 1. Критичные баги (🔴)

- [ ] **T1. Автогенерация → 502.** Перевести UI на уже существующий асинхронный запуск
  (`202 Accepted` + опрос статуса/прогресса), поднять `proxy_read_timeout`/`proxy_send_timeout`
  в nginx. Бэкенд-очередь генерации уже есть.
  _Файлы: `Frontend/src/components/ScheduleTab.vue`, `Frontend/src/api/lessons.ts`,
  `Frontend/nginx.conf` (бэк — `Schedule/Generation`, эндпоинт `generate async/status`)._

- [ ] **T2. Ограничения не сохраняются.** Любые вводимые ограничения не персистятся. Сделать
  сохранение через API и убедиться, что данные доходят до солвера:
  доступность преподавателей/аудиторий (сетка дни×пары), привязка оборудования к аудиториям и
  планам, прочие правки параметров плана.
  _Файлы: фронт — вкладка «Ограничения» (`SettingsTab.vue`, `TimeGrid.vue`), `api/constraints.ts`,
  `api/management.ts`; бэк — эндпоинты доступности/оборудования, репозитории, маппинг в `ScheduleModel`._

- [ ] **T3. Двойная пара не ставится.** Диагностировать `DoubleLessonSectionBuilder` и маппинг
  `Curriculum.Double` (почему занятие не размещается двумя смежными слотами) и починить.
  _Файлы: `Application/Solver/Builder/BuildSections/DoubleLessonSectionBuilder.cs`, маппер плана._
